# Документация проекта: Модульный конструктор сайтов (Vue 3 + TS)

## 1. Общее описание

Проект представляет собой интерактивный стенд для динамической сборки лендингов. Система основана на принципе **Data-Driven UI**: визуальное отображение сайта полностью диктуется JSON-структурой, хранящейся в реактивном объекте `mySiteObject`.

## 2. Архитектура данных

Данные организованы в двухуровневую иерархию: **Ряды (Rows) → Блоки (Blocks)**.

### Финальная структура `mySiteObject`

```typescript
const mySiteObject = ref([
  // РЯД №1: Текстовая секция
  {
    rowId: "row_1",
    rowTitle: "Верхняя секция",
    blocks: [
      { 
        id: "B1", 
        type: "text", 
        title: "Текст 1", 
        data: { content: "Я в первом ряду!" } 
      }
    ]
  },
  
  // РЯД №2: Секция с медиа-контентом
  {
    rowId: "row_2",
    rowTitle: "Средняя секция (Картинки)",
    blocks: [
      { 
        id: "B2", 
        type: "image", 
        title: "Фото 1", 
        data: { src: img1 } 
      },
      { 
        id: "B3", 
        type: "image", 
        title: "Фото 2", 
        data: { src: img2 } 
      }
    ]
  },

  // РЯД №3: Подвал или завершающая секция
  {
    rowId: "row_3",
    rowTitle: "Нижняя секция",
    blocks: [
      { 
        id: "B4", 
        type: "text", 
        title: "Заголовок", 
        data: { content: "Конец страницы" } 
      }
    ]
  }
]);

```

## 3. Реализованные процессы разработки

### Сброс стандартных ограничений (Global Styles)

Для работы во всю ширину экрана были переписаны стили, генерируемые Vue по умолчанию (`main.css`):

* Удалено ограничение `max-width: 1280px`.
* Отключено центрирование `margin: 0 auto`.
* Исправлен медиа-запрос на `1024px`, который принудительно включал режим `grid`.
* Установлено `width: 100%` для всех корневых контейнеров (`html`, `body`, `#app`).

### Механизм динамического рендеринга

В главном холсте (`SitePreview.vue`) реализован двойной цикл `v-for`:

1. **Внешний цикл** создает контейнеры для рядов (`.layout-row`).
2. **Внутренний цикл** рендерит блоки через системный компонент `<component :is="..." />`.

Связь типов блоков с Vue-компонентами реализована через карту соответствия:

```typescript
const componentsMap = {
  text: TextBlock,
  image: ImageBlock
};

```

### Передача данных (Props Flow)

Использован паттерн **Props Spreading**. При вызове компонента через `v-bind="block.data"`, Vue автоматически деструктурирует объект:

* Ключ `content` внутри `data` становится пропсом `content` в `TextBlock.vue`.
* Ключ `src` становится пропсом `src` в `ImageBlock.vue`.

## 4. Стилизация интерфейса

* **Layout**: Блоки внутри ряда выстраиваются горизонтально (`display: flex`) с возможностью переноса (`flex-wrap: wrap`).
* **Размеры**: Каждый блок имеет фиксированный размер **225x225px** (увеличено на 50% от базового).
* **Дизайн карточек**:
* Заголовки блоков вынесены в отдельную «шапку» с фоном `#f9fafb`.
* Добавлены скругления `border-radius: 12px` и тени.
* Реализован Hover-эффект: при наведении блок приподнимается (`translateY(-4px)`).


* **Изображения**: Применяется `object-fit: contain` и внутренние отступы (`padding`), что предотвращает прилипание картинки к рамкам блока.

## 5. Инструкция по эксплуатации

Для добавления нового блока или ряда достаточно изменить состояние `mySiteObject`. Система автоматически перерисует интерфейс без перезагрузки страницы.
